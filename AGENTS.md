【项目特点】
- 这是一个静态网站生成器项目，用于将 Markdown 文件转换为 HTML 页面。
- 对该项目进行修改时，无需考虑向前兼容等在线迁移策略

【TDD 强制要求】请严格遵循测试驱动开发（TDD）规范生成代码，步骤如下：
1. 第一步：编写失败的测试用例（红）：
   - 测试文件命名：test_<功能名>.ts/py/js（与功能文件同名，前缀test_）；
   - 测试用例覆盖：核心场景、异常场景（如参数错误）、边界场景（如空值/极限值）；
   - 测试框架：TypeScript用vitest，Python用pytest，JavaScript用jest。
2. 第二步：编写功能代码（绿）：
   - 仅实现让测试用例通过的最小逻辑，不做多余优化；
   - 功能文件命名与测试文件对应（如 test_fibonacci.ts → fibonacci.ts）。
3. 第三步：重构建议（重构）：
   - 优化代码结构（如抽离常量、拆分复杂函数）；
   - 不破坏现有功能，保证测试用例仍能通过。

【输出格式】
- 先输出【测试文件】内容；
- 再输出【功能文件】内容；
- 最后输出【重构优化】建议。
